# 11. Phase 4 重複・ReviewQueue・管理者判断

実装・確認日: 2026年7月4日

## 結論

重複候補の採点、ReviewQueueへの優先度付き振り分け、管理者判断、追記専用の監査履歴、明示承認型のAutomationRulesを実装しました。

高い類似度だけではイベントを自動統合しません。公式URL、申込URL、PDFハッシュのいずれかが一致する場合だけ「自動統合候補」にでき、最終的な統合判断とcanonical eventの指定は管理者が行います。

## 重複判定

初期スコアは設計どおり次の重みです。

| 特徴 | 重み |
|---|---:|
| 正規化タイトル | 0.40 |
| 開催日 | 0.25 |
| 主催団体 | 0.15 |
| URL・PDFハッシュ | 0.15 |
| 開催形式 | 0.05 |

正規化ではNFKC、空白、記号、Web／オンライン等の形式語を比較用タイトルから除きます。「第N回」「年度」は削除しないため、別開催回を同一イベントと誤認しにくくしています。主催者比較では法人格だけを除きます。

判定:

- 0.85以上かつ決定的アンカーあり: 自動統合候補
- 0.65以上: DuplicateCandidatesで管理者確認
- 0.65未満: 原則別イベント

「自動統合候補」は自動実行ではありません。管理者が`重複統合`を確定するまで、元イベントと全掲載元を保持します。

## ReviewQueue

優先度:

1. 申込締切まで72時間以内: 緊急
2. 公開中イベントの重要項目変更: 高
3. 開催まで7日以内: 高
4. 通常の新規・不確実候補: 通常
5. すでに終了した候補: 低

提案アクション:

- 重複候補: 統合
- イベント性が低い: 非公開
- 日時、締切、料金、単位が不確実: 修正
- その他の不確実項目: 要確認付き公開

根拠抜粋は空白を正規化して300文字以下にし、PDF全文やAI生応答をReviewQueueへ複製しません。

## 管理者判断

対応する判断:

- 公開
- 要確認付き公開
- 修正後公開
- 非公開
- 重複統合

判断時には変更前後をJSONでReviewActionsへ追記します。元イベントを削除せず、重複統合では次を設定します。

- 元イベント: 非公開
- `duplicate_status`: 統合済み
- `canonical_event_id`: 管理者が指定した統合先

FieldOverridesとEventSourcesをcanonical eventへ移す処理は、Sheets接続後の一括更新層からこの判断結果を使って実行します。元の監査行は残します。

## AutomationRulesの安全制約

`次回も確認`は、内部フラグが誤って有効でもAutomationRulesを作りません。

ルール作成に必要な条件:

1. automation_choiceが自動公開、要確認付き公開、非公開のいずれか
2. 管理者がルール作成を明示承認
3. 空でない適用条件
4. 元ReviewActionのID、承認者、承認日時を保存

GASではEventSourcesからsource_idを取得し、掲載元単位の条件を作ります。source_idを特定できない場合はルールを作りません。

## Google Apps Script

[Code.gs](../gas/Code.gs)を非公開の管理スプレッドシートへ紐付けると、`CE Seminar Finder`メニューが追加されます。

主な操作:

- ReviewQueueで選択した行の判断確定
- Eventsの公開・レビュー・統合状態更新
- ReviewActionsへの変更前後追記
- 明示承認された場合だけAutomationRulesへ追記
- 必須シート構造の確認

同時実行はDocumentLockで30秒まで待機します。判断途中の競合を避け、ReviewActionsは更新せず追記だけ行います。

重複統合ではReviewQueueの`decision_note`に次の形式で統合先を記載します。

```text
canonical_event_id=evt_xxxxxxxxxxxx
```
