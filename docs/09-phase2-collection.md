# 9. Phase 2 非AI収集の実装結果

実装・確認日: 2026年7月4日

## 結論

公式9団体に対する非AI収集基盤を実装しました。8団体は保存済み実レスポンスからイベント候補を抽出でき、沖縄はクライアント側描画が必要であることを検出して、安全に技術的保留へ移します。1団体の失敗や保留で、他団体の処理は停止しません。

本番用のSources初期値はすべて無効のままです。連絡先、Google Sheets接続、日次運用条件が揃う前に自動巡回を開始しない設計です。

## 実装したもの

- HTTP/HTTPS以外、認証情報付きURL、許可外ホスト・パスの拒否
- DNS解決後のプライベート、ループバック、リンクローカルIP拒否
- リダイレクト先の再検査と回数制限
- robots.txtの拒否規則と`Crawl-delay`の適用
- ホスト単位の取得間隔
- UMINの最低60秒間隔
- ETag、Last-Modified、本文SHA-256による無変更判定
- 408、429、5xxだけを対象にした上限付き再試行
- 最大レスポンスサイズと1実行あたり取得件数の上限
- CP932のmeta charset検出
- 9団体の独立アダプター
- 候補、警告、取得ログのJSONL保存
- Google SheetsのEventSources、FetchLogsへの追記口
- 1 URLの失敗後も次の入口を処理するパイプライン

## 実サイト確認の範囲

同一サイトへ反復アクセスせず、登録済みの代表入口を各1回だけ取得してアダプターを確認しました。確認用レスポンスは`var/`配下に置き、リポジトリや公開成果物へ含めません。

ローカル環境の認証局設定により通常の証明書検証ができなかったサイト確認では、調査用の単発取得だけ証明書検証を迂回しました。製品コードのHTTPクライアントは証明書検証を無効化せず、この迂回設定も持ちません。

## 沖縄の扱い

初期HTMLはNuxtのローディング用シェルだけで、イベント本文がありません。現段階では推測した非公開APIへアクセスせず、次の順序で後続対応します。

1. 公開ブラウザ通信から公式の公開APIを確認
2. APIがなければ、1日1回・最小ページ数のブラウザ取得
3. 安定しなければ日本臨床工学技士会等の掲載から候補を補完
4. `SOURCE_GAP`をReviewQueueに残し、自動公開しない

## 実行例

全テスト:

```bash
PYTHONPATH=src python3 -m unittest discover -s tests -v
```

無効状態のSourceを調査目的で1件だけ実行:

```bash
PYTHONPATH=src python3 -m ce_seminar_finder.cli collect-source \
  --source-id src_fukuoka \
  --force \
  --max-detail-pages 0
```

通常運用では`--force`を使わず、非公開SheetsのSourcesで明示的に有効化した対象だけを処理します。

## テスト範囲

36件の自動テストで、少なくとも次を確認します。

- 9団体すべてにアダプターがある
- RSS、HTML、CP932、見出しブロック、イベント表の候補抽出
- 空文字PDFリンク、対象外表、ナビゲーション、URLタイトルの除外
- Nuxtシェルの技術的保留
- robots拒否時に本文を取得しない
- UMIN相当の取得間隔を待つ
- DNS、許可ホスト・パス、認証情報付きURLの拒否
- 許可外リダイレクトの拒否
- 条件付き取得と本文ハッシュによる無変更スキップ
- 再試行対象の限定
- 1 URLの失敗で全体を止めない

## Phase 3への引き継ぎ

大分の大量の過去イベント、鹿児島の事務連絡混在、全国情報と県会情報の重複は、Phase 2で無理に削除しません。原文とURLを保持した候補として受け渡し、Phase 3の日時・イベント性抽出とPhase 4のReviewQueue・重複判定で扱います。
