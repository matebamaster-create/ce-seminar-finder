# 12. Phase 5 公開サイトMVP

実装日: 2026年7月4日

## 状態

公開許可済みJSONだけから生成する静的サイトMVPを実装しました。トップ、イベント一覧、詳細、アーカイブ、説明ページを生成できます。

ローカルファイル・localhostへのブラウザ接続が実行環境の安全ポリシーで拒否されたため、実ブラウザによる画面受入確認はPhase 6の公開URLで再実行します。HTML、リンク、ID、JavaScript構文、公開データ境界、レスポンシブCSSはローカル検査済みです。

## ページ

- `/`
- `/events/`
- `/events/{event_id}/`
- `/archive/`
- `/about/`

イベント名称をURLに使わず、永続event_idを詳細URLに使います。ページ・CSS・JavaScriptの参照は相対パスのため、GitHub Pagesのサブパス配信にも対応します。

## トップ

- Web、オンデマンド、ハイブリッドを優先した新着欄
- キーワード検索
- 8ジャンルへの導線
- 5開催形式への導線
- 公式確認を促すフッター

サンプル件数が少ない場合も空状態を表示し、レイアウトを崩しません。

## 一覧・アーカイブ

検索条件:

- キーワード
- ジャンル
- 開催形式
- イベント種別
- 参加費
- 開催月
- 申込受付中のみ

並び替え:

- 開催日が近い順
- 申込締切が近い順
- イベント名順

検索件数と選択条件は`aria-live`で通知します。全解除後はキーワード欄へフォーカスを戻します。JavaScriptが無効でも、生成時点のカードと公式リンクはHTML内に残ります。

## 詳細

- 形式、ジャンル、種別
- 開催日時、配信期間、申込締切
- 料金、対象、定員、会場、単位
- 公式詳細を最優先した外部リンク
- 申込、公式PDFへの補助リンク
- 要確認理由
- 公式情報確認の注意書き

外部リンクは新しいタブで開くことをアクセシブルな名前で伝え、`rel="noopener"`を付与します。

## PDF検索

PDF全文や抽出チャンクは公開しません。管理側で承認した非機微な検索語だけをNFKC正規化し、2文字・3文字n-gramのSHA-256へ変換します。

公開JSON:

- event_id
- `pdf_search_hashes`

ブラウザ:

1. 2文字以上の検索語を同じ方法で正規化
2. Web Cryptoでn-gramをSHA-256化
3. 全ハッシュが一致したイベントを表示
4. 通常フィールドに一致せずPDFだけ一致した場合、「PDF内に関連語あり」を表示

承認済み検索語そのもの、PDF本文、AI入力は公開しません。

## デザイン・アクセシビリティ

- 青緑、ネイビー、白、淡いグレーのトークン
- 本文16px基準
- 44px以上の主要操作領域
- 320px以上の画面を対象
- 820pxと520pxで1列レイアウトへ変更
- セマンティックな見出し、`dl`、`time`
- 本文へのスキップリンク
- 見えるフォーカスリング
- 色だけに依存しない状態ラベル
- `prefers-reduced-motion`
- 限定共有用`noindex,nofollow`

## 生成

```bash
python3 -m ce_seminar_finder.cli build-site \
  --data site/data/events.json \
  --output-directory site
```

生成結果には、公開許可済みのcanonical eventだけを含む`site/data/events.json`を使用します。

## 検証

全67件のテストが成功しています。サイト関連では次を検査します。

- 現在イベントと過去イベントの分離
- 全5種類のページ生成
- HTMLエスケープ
- 非公開項目の非混入
- PDF検索語を公開せずハッシュだけを出力
- `noindex`、スキップリンク、`aria-live`、`noopener`
- 生成済み全HTMLのローカルリンク切れ
- HTML内IDの重複
- JavaScript構文
- Pythonコンパイル

GitHub Actionsでも静的サイトを再生成し、`admin_note`、`auto_values`、`FieldOverrides`、`ReviewQueue`が成果物へ混入していないことを確認します。
