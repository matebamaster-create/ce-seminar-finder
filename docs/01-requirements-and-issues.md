# 1. 要件要約と重要な設計論点

## 要件の要約

CE Seminar Finderは、都道府県臨床工学技士会と日本臨床工学技士会の公式サイトに分散する学習機会を収集し、Web・オンデマンド・ハイブリッド開催を見つけやすくする発見サービスです。

初期版は九州8県と日本臨床工学技士会を対象にし、次を一連の運用として成立させます。

1. 公式サイトの候補ページ、記事、PDFを1日1回以下で取得する
2. ルールとAIでイベント候補、構造化項目、分類を抽出する
3. 同一イベントの複数掲載を検出して統合候補を作る
4. 不確実な情報をReviewQueueに送り、管理者が確認する
5. 管理者修正値を優先して公開用データを生成する
6. スマートフォン中心の静的サイトで検索、絞り込み、詳細表示を提供する
7. 終了イベントは通常一覧から外し、アーカイブに残す

対象には技士会主催だけでなく、技士会公式サイトに掲載された関連団体、企業主催、企業共催のイベントも含めます。求人、純広告、物販、会費、選挙、事務連絡等は対象外ですが、判定が曖昧なものは削除せずReviewQueueに保持します。

## 重要な設計上の論点

### 1. 「掲載元」と「主催者」は別物

技士会サイトが他団体のイベントを紹介することが多いため、`source_organization`（掲載元）と`organizer_name`（主催者）を分けます。URLも単一列に押し込まず、主催者公式URL、申込URL、PDF URL、掲載元URLを別管理します。

メインの公式詳細URLは次の優先順位で決めます。

1. 主催団体の公式詳細ページ
2. 主催団体の公式PDF
3. 日本臨床工学技士会の掲載ページ
4. 都道府県技士会の掲載ページ
5. 外部申込サービス

### 2. 自動取得値、管理者修正値、表示値

3値を同じセルに上書きすると、再取得で管理者修正が消えます。初期版から次を分離します。

- 自動取得値: AI・ルールの出力、信頼度、根拠を保持
- 管理者修正値: 誰が、いつ、なぜ変更したかを保持
- 表示用最終値: `管理者修正値 > 採用済み自動値 > 空欄` で生成

Eventsシートには日常運用向けの最終値を表示し、詳細な来歴は正規化した補助シートに持たせます。

### 3. 自動公開は厳格な許可制

次の全条件を満たした候補だけを自動公開対象にします。

- イベント性が高信頼
- イベント名、開催日または配信期間、主催団体、開催形式、公式詳細URLが明確
- 申込締切、参加費、単位・ポイントが「明記あり」「該当なし」のいずれかで、不明ではない
- 重複候補が未解決ではない
- 抽出元が到達可能
- サイト別ルールが自動公開を許可
- 公開終了日を計算できる

要件どおり、締切、参加費、単位情報等が不明な場合はReviewQueueへ送ります。このため初期運用はReviewQueueが多くなる見込みです。運用実績を見て、管理者が明示許可したパターンだけをAutomationRulesへ昇格させます。

### 4. PDF全文は1セルに保存しない

Googleスプレッドシートは1セルの文字数に上限があるため、PDF全文をEventsの1セルへ保存する設計は避けます。文書単位のメタデータをDocumentsに、抽出テキストを40,000文字以下のチャンクとしてDocumentTextChunksに保存します。公開サイトには全文を出しません。

### 5. 日付は単一の「開催日」にしない

次を別々に持ちます。

- 現地・ライブの開始日時と終了日時
- オンデマンド配信開始日と終了日
- 申込開始日時と締切日時
- 公開終了日
- 日時の精度（日、時刻まで、月のみ、不明）
- タイムゾーン（初期値はAsia/Tokyo）

「ライブ＋後日オンデマンド」は開催形式をハイブリッドと決めつけず、`live_format`と`has_on_demand`を併記して最終ラベルを決定します。

### 6. 重複はイベントを消す処理ではなく、情報源を束ねる処理

EventSourcesで複数の掲載元を保持し、主イベントに統合しても元URL、取得日時、抽出根拠を失わないようにします。AIの類似判定だけで自動統合せず、URL一致や日付・主催者・正規化タイトルの強い一致を必要とします。

### 7. サイトごとの差が大きい

調査時点で次の差異があります。

- 福岡、日本臨床工学技士会: WordPress、RSS・サイトマップあり
- 佐賀、長崎、大分: 静的または独自HTML、一覧構造が比較的明確
- 熊本: Shift_JIS、HTTP中心、HTTPS接続が不安定
- 宮崎: Wixの動的ページ
- 鹿児島: WordPressだがカスタム投稿とクエリ形式
- 沖縄: JavaScriptアプリ、本文はクライアント取得、サイトマップのホスト名に不整合

共通クローラーだけではなく、宣言的なサイト設定と最小限のサイト別アダプターが必要です。

### 8. APIキーと非公開情報の境界

- GitHub Actions Secrets: AI API、Googleサービスアカウント、必要な配布トークン
- GAS Script Properties: GitHub手動実行用トークン等
- Googleスプレッドシート: 非公開、閲覧者を限定
- GitHub Pages: 公開許可済みの最小データだけ
- ブラウザJavaScript: APIキー、スプレッドシートID、管理用URLを置かない

### 9. 検索は静的サイト内で完結させる

MVPは公開イベントをJSONとしてビルド時に生成し、ブラウザ内インデックスで検索します。ユーザー登録や個人情報を扱わず、サーバーDBも不要です。PDF本文はインデックス用の語彙に変換し、公開JSONへ全文を含めません。

### 10. 法務・運用は機械判定で完結しない

robots.txtはアクセス許可の包括的な法的承諾ではありません。各団体の利用規約・サイトポリシー、削除依頼、取得停止依頼を尊重し、Sourcesシートの`enabled`をサイト単位で即時停止できるようにします。転載は必要最小限の事実情報に限定し、公式リンクを主導線にします。

## MVPで扱わないもの

- 一般ユーザーのログイン、個人設定、お気に入り、通知
- 申込代行、決済、参加者情報の保持
- 広告、スポンサー枠、有料掲載
- 本格的な管理者Webアプリ
- 機械学習モデルの再学習
- 全国47都道府県への同時展開

## 成功指標の初案

- 対象9団体すべてについて日次取得の成否が記録される
- 取得成功サイトでは新規候補が24時間以内にEventsまたはReviewQueueへ入る
- 公開イベントの100%に公式詳細URLまたは公式PDF URLがある
- 管理者修正済みフィールドの自動上書きが0件
- 重複統合後も全掲載元URLを追跡できる
- 公開サイトでWeb系イベントが初期表示の上位に出る
- スマートフォン幅360pxで主要操作が横スクロールなしに完了する
